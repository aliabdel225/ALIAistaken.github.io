Project Overview

This investigation examined suspicious Windows activity associated with Stuxbot behavior. The analysis focused on tools staged in C:\Users\Public, registry-based persistence, and PowerShell Remoting used for lateral movement.

Key Findings

FindingEvidenceWhy It Matters
Tool stagingExecutables and utilities appeared under C:\Users\Public.Public directories are frequently abused because they are writable and accessible.
PersistenceRegistry Run keys referenced suspicious programs.Run keys can automatically launch malware when a user signs in.
Lateral movementPowerShell Remoting activity targeted additional systems.Remote PowerShell may indicate movement toward privileged hosts.

MITRE ATT&CK Mapping

  • T1105 — Ingress Tool Transfer
  • T1060 / T1547.001 — Registry Run Keys
  • T1021.006 — Windows Remote Management
Analyst Note: The goal of this project was not only to find suspicious activity, but to explain how the evidence supports the conclusion and what defenders should do next.

Lessons Learned

  • Correlating multiple log sources produces stronger conclusions than relying on one event.
  • File paths, parent processes, command lines, and user context should be reviewed together.
  • Threat-hunting reports should include both evidence and response recommendations.

“Every security event tells a story. The analyst’s job is to preserve it, understand it, and report it clearly.”
